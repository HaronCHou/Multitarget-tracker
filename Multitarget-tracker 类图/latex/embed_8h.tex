{\footnotesize\ttfamily void finalize\+\_\+interpreter (\begin{DoxyParamCaption}{ }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [inline]}}

\textbackslash{}rst Shut down the Python interpreter. No \mbox{\hyperlink{namespacepybind11}{pybind11}} or CPython API functions can be called after this. In addition, \mbox{\hyperlink{namespacepybind11}{pybind11}} objects must not outlive the interpreter\+:

.. code-\/block\+:: cpp \begin{DoxyVerb}{ // BAD
    py::initialize_interpreter();
    auto hello = py::str("Hello, World!");
    py::finalize_interpreter();
} // <-- BOOM, hello's destructor is called after interpreter shutdown

{ // GOOD
    py::initialize_interpreter();
    { // scoped
        auto hello = py::str("Hello, World!");
    } // <-- OK, hello is cleaned up properly
    py::finalize_interpreter();
}

{ // BETTER
    py::scoped_interpreter guard{};
    auto hello = py::str("Hello, World!");
}
\end{DoxyVerb}
 .. warning\+:\+: \begin{DoxyVerb}The interpreter can be restarted by calling `initialize_interpreter` again.
Modules created using pybind11 can be safely re-initialized. However, Python
itself cannot completely unload binary extension modules and there are several
caveats with regard to interpreter restarting. All the details can be found
in the CPython documentation. In short, not all interpreter memory may be
freed, either due to reference cycles or user-created global data.
\end{DoxyVerb}

\doxysubsubsection{\texorpdfstring{initialize\_interpreter()}{initialize\_interpreter()}}
{\footnotesize\ttfamily void initialize\+\_\+interpreter (\begin{DoxyParamCaption}\item[{bool}]{init\+\_\+signal\+\_\+handlers = {\ttfamily true} }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [inline]}}

\textbackslash{}rst Initialize the Python interpreter. No other \mbox{\hyperlink{namespacepybind11}{pybind11}} or CPython API functions can be called before this is done; with the exception of {\ttfamily PYBIND11\+\_\+\+EMBEDDED\+\_\+\+MODULE}. The optional parameter can be used to skip the registration of signal handlers (see the {\ttfamily Python documentation}\+\_\+ for details). Calling this function again after the interpreter has already been initialized is a fatal error.

If initializing the Python interpreter fails, then the program is terminated. (This is controlled by the CPython runtime and is an exception to \mbox{\hyperlink{namespacepybind11}{pybind11}}\textquotesingle{}s normal behavior of throwing exceptions on errors.)
